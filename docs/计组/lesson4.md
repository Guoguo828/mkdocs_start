### 第一层：计算机的核心组成

计算机由三个核心部分构成：

- **CPU（中央处理器）**：负责指令执行、数据运算与控制。
- **Memory（内存）**：存储程序和数据。
- **I/O interface（输入输出接口）**：连接外部设备（如键盘、显示器、硬盘等）与计算机核心部件。

### 第二层：CPU的内部构成

CPU由两个关键模块组成：

- **Control unit（控制单元）**：负责生成控制信号，协调CPU内部及计算机各部件的操作（如指令的读取、解码、执行时序）。
- **Datapath（数据通路）**：是数据在CPU内部传输、运算的物理路径，包含以下组件：
  - **Multiplexers（多路选择器）**：选择数据的传输路径。
  - **ALU（算术逻辑单元）**：执行算术运算（如加减）和逻辑运算（如与或非）。
  - **Registers（寄存器）**：临时存储数据和指令，是CPU内部的高速存储单元。

## 指令执行的overview

### 1. 所有指令的前两步完全相同

- **取指（Fetch the instruction from the memory）**：从内存中读取要执行的指令。
- **解码并读寄存器（Decode and read the registers）**：解析指令含义，并从寄存器中读取所需的操作数。

### 2. 后续步骤由指令类型决定

指令分为三类，后续流程因类型而异：

- 内存引用型（Memory-reference，如`ld`加载、`sd`存储）
- 算术逻辑型（Arithmetic-logical，如`add`、`and`等运算指令）
- 分支型（branches，如`beq`条件跳转）

### 3. 按指令类型的具体执行动作

根据指令类别，会执行以下操作：

- **使用ALU计算**：
  - 算术结果（如加减运算的结果）；
  - 加载/存储的内存地址；
  - 分支指令的比较逻辑（如`beq`的相等判断）。
- **加载/存储指令访问数据内存**：完成数据的读写操作。
- **更新程序计数器（PC）**：跳转到目标地址（分支指令）或执行下一条指令（PC + 4）。

#### 7个控制信号是什么

| 信号名称          | 无效时（=0）的效果                                       | 有效时（=1）的效果                                                                 |
|-------------------|----------------------------------------------------------|----------------------------------------------------------------------------------|
| 寄存器写（RegWrite） | 无                                                       | 寄存器目的输入被写入“写数据输入”上的值                                             |
| ALU源（ALUScr）    | 第二个ALU操作数来自第二个寄存器文件输出（读数据2）| 第二个ALU操作数是指令低16位符号扩展后的值                                         |
| 分支（PCSrc）| PC被计算“PC+4”的加法器输出替换| PC被计算分支目标的加法器输出替换                                                   |
| 跳转（Jump）| PC被PC+4或分支目标替换| PC由加法器计算的跳转地址更新                                                       |
| 存储器读（MemRead） | 无                                                       | 由地址输入指定的数据存储器内容被放到“读数据输出”上                                 |
| 存储器写（MemWrite） | 无                                                       | 由地址输入指定的数据存储器内容被“写数据输入”上的值替换                             |
| 存储器到寄存器（2位）（MemtoReg） | 00：送入寄存器“写数据输入”的值来自ALU| 01：送入寄存器“写数据输入”的值来自数据存储器 <br> 10：送入寄存器“写数据输入”的值来自PC+4 |

**这些控制信号都是输入多路选择器 或者寄存器 或者内存的

#### ALU用途及控制

- **加载/存储（Load/Store）**：功能为加法（F = add）
- **分支（Branch）**：功能为减法（F = subtract）
- **R型指令**：功能取决于操作码（F depends on opcode）

| ALU 控制 | 功能       |
|----------|------------|
| 0000     | AND        |
| 0001     | OR         |
| 0010     | add        |
| 0110     | subtract   |
