    指令：Language of the Machine¶

    介绍¶

    指令有指令集 我们这里讲的是RISC-V指令集 全称是Reduced Instruction Set Computer

    设计目标是提高表现 降低消费 减少设计时间

    指令都是由操作符（add）还有操作数（a b c）组成

    影响指令特性的关键因素有以上五条

    计算机硬件的操作¶

    stored program concept¶

    所有的东西存下来都是以数字形式的（01）

    原则¶

    1.一个指令只能有一个操作（只能加 或者 只能减） 2.只能有三个变量（操作数）

    内存命令符¶

    1. 内存用于存储复合数据¶

    主内存（Main Memory）适合存放“复合数据”，比如：

    • 数组（Arrays）：多个相同类型元素的集合（如int arr[100]）；
    • 结构体（Structures）：多种类型元素的组合（如包含int、float的自定义结构）；
    • 动态数据（Dynamic data）：堆上分配的、运行时才确定大小的数据（如C++的new、Python的列表）。 → 原因：寄存器容量小（仅几十个），而内存容量大，能承载体积更大、更复杂的数据结构。

    2. 算术运算与内存的交互¶

    CPU的算术运算（如加减乘除） 主要在寄存器中进行（寄存器速度远快于内存）。因此，数据需要在“内存”和“寄存器”间搬运：

    • 先通过 Load指令：把内存中的数据加载到寄存器；
    • 运算完成后，通过 Store指令：把寄存器中的结果写回内存。

    3. 内存的寻址方式：按字节寻址（Byte Addressed）¶

    内存的每个地址唯一对应一个 8位的字节（Byte）。也就是说，内存的“寻址粒度”是1字节——地址空间里的每个“单元”都是1字节，这是内存访问的基本规则。

    4. 字节序：小端序（Little Endian）¶

    RISC-V采用小端序存储多字节数据（如“字（Word，通常4/8字节）”）。“小端序”的规则是：

    • 一个多字节数据中，最低有效字节（LSB，Least-Significant Byte） 存放在该数据的最低地址处。
    • 对比“大端序（Big Endian）”：大端序中，最高有效字节（MSB，Most-Significant Byte） 存放在最低地址处。

    → 举例：存储数值 0x12345678（4字节）：

    • 小端序：地址从低到高，依次存 78、56、34、12；
    • 大端序：地址从低到高，依次存 12、34、56、78。

    5. 内存对齐要求：无强制对齐¶

    RISC-V不要求“字（Word）”在内存中必须“对齐”存储（“对齐”指：数据的起始地址是其长度的整数倍。比如4字节的字，对齐的地址是0x...0、0x...4等4的倍数）。 → 对比：有些其他指令集架构（如早期ARM、MIPS）会强制内存对齐，否则触发错误或性能损失。RISC-V的设计更灵活，但非对齐访问可能需要额外硬件逻辑处理。

    计算机中指令的表示¶

    简介¶

    1. 信息的二进制本质：计算机中所有信息（包括指令、数据）都由二进制位（bits）组成。
    2. 指令的二进制编码：指令会被编码为二进制形式，称为机器码（machine code），是CPU能直接识别执行的格式。
    3. 寄存器的数字映射：RISC-V的32个通用寄存器（x0~x31）被映射为数字0~31，便于指令编码时表示寄存器编号。
    4. RISC-V指令的编码特性：
    5. 每条指令被编码为32位的指令字（固定长度，简化硬件解码）。
    6. 采用少量统一的格式，编码“操作码（opcode，指定指令类型）”“寄存器编号”等关键信息。
    7. 设计具有规整性（Regularity），让硬件解码更高效，也便于编译器生成/优化代码。

    基本块¶

    1. 基本块的定义与特征

    基本块是一段指令序列，具有两个关键特征： 「无嵌入式分支（末尾除外）」：在基本块内部不会出现分支指令（如跳转、条件分支等），仅在块的末尾可能存在分支操作。 「无分支目标 / 分支标签（开头除外）」：只有基本块的开头可以作为分支指令的目标位置（即被跳转指令指向），块内部不能有分支标签或成为分支目标。

    1. 基本块的作用 对于编译器：识别基本块是编译优化的基础。编译器可以基于基本块进行各类优化操作（如常量传播、死代码消除、指令重排等），提升代码执行效率。 对于处理器：先进的处理器可通过指令级并行等技术，加速基本块的执行，从而提升程序运行速度。

    函数的定义和步骤¶

    函数是什么：就是一段程序，基于传入参数执行任务

    步骤

    1.参数传递：将参数放置在过程（函数）可访问的位置，例如 MIPS 架构通过函数参数寄存器a0-a7 传递参数。 2.控制转移：通过 “跳转” 操作，将程序控制权转移到过程的入口。 3.获取存储资源：为过程分配所需的存储资源（如栈空间，用于保存局部变量、寄存器值等）。 4.执行任务：运行过程内部逻辑，完成特定功能。 5.结果返回：将结果放置在调用者程序可访问的位置，例如 MIPS 架构通过返回值寄存器a0-a1 传递结果。 6.控制返回：将程序控制权返回给调用点，通过返回地址寄存器ra 定位调用时的原位置。

    帧指针¶

    先看一下不同东西存的结构

    这里的帧指针的作用是什么？ 因为栈指针一直指的是栈顶 会变动 不方便加偏移量使用

    而帧指针是不变的 一直指的是那些saved的东西的第一个（这样方便偏移查询）

    1. 核心主题¶

    讲解函数调用时，如何在栈里为函数分配空间（存局部变量、保存的寄存器等数据）。

    2. 关键概念¶

    • 过程帧/活动记录（Procedure frame/activation record） 栈里的一段“专属区域”，用来存某个函数的：

    • 被保存的寄存器（比如之前说的s寄存器）；

    • 函数的局部变量（这不是寄存器）。 简单说：每个函数调用时，栈都会单独划一块空间存它的临时数据，这就是“过程帧”。

    • 帧指针（Frame pointer，简称FP） 一个“标记”（存在寄存器里，比如RISC-V的s0/fp），用来定位当前函数的过程帧位置——通过FP能快速找到这个函数的局部变量、保存的寄存器存在栈的哪里。

    • 被调用方分配的局部数据 比如C语言里的“自动变量”（就是函数内的局部变量，比如int x;），这些数据是由“被调用的函数”（callee）在栈上分配空间的。

    3. 栈帧的结构（右边的图）¶

    栈是“高地址在上、低地址在下”（栈向下增长），图里的SP是栈指针（指向栈顶），FP是帧指针（指向当前函数栈帧的起始位置）。 栈帧里通常会存这些内容：

    • 保存的参数寄存器；
    • 保存的返回地址；
    • 保存的s寄存器；
    • 局部数组、结构体等数据（因为寄存器放不下）。

    内存的布局¶

    这里写的很明白了。

    这页PPT讲的是程序运行时的“内存布局”——也就是程序在内存里被分成了哪几个功能区域，每个区域存什么数据，拆解如下：

    1. 内存的4个核心区域¶

    程序运行时，内存会被划分为4个独立的“段”，每个段有明确的用途：

    • Text段：存程序的代码（指令），比如你写的函数逻辑对应的机器指令，PC（程序计数器）会指向这个段的地址来执行代码。

    • Static data段：存全局变量、静态变量（比如C里的static int a;、常量数组、字符串常量）。 这里还提到RISC-V的x3（全局指针gp）——它会被初始化为这个段的地址，方便快速访问全局变量。

    • Dynamic data段（堆）：存动态分配的数据（比如C里malloc申请的内存、Java里new创建的对象），需要手动申请/释放。

    • Stack段（栈）：存自动变量（也就是函数的局部变量），随函数调用自动分配/销毁（之前讲的栈帧就在这里）。

    2. 内存地址的分布（右边的图）¶

    内存地址是“从低到高”排列的：

    • 最底部是Reserved（保留区域）；
    • 往上是Text（程序代码，PC指针指向这里）；
    • 再往上是Static data（全局/静态变量）；
    • 然后是Dynamic data（堆）（堆是向上增长的）；
    • 最顶部是Stack（栈）（栈是向下增长的）。 （堆和栈中间的区域，是它们动态扩展的空间）

    3. C变量的“存储类”对应关系¶

    C里的变量类型，会对应到不同的内存段：

    • automatic（自动变量）：对应Stack段（函数局部变量）；

    synchronization in RISC-V¶

    讲的是RISC-V架构中的同步机制

    1. 多处理器共享内存的竞争问题¶

    当两个处理器（P1、P2）共享一块内存区域时：

    • 若执行“P1写内存 → P2读内存”的操作，若不同步会引发数据竞争（Data race）；
    • 数据竞争的结果会依赖于访问顺序，导致程序行为不可预测。

    2. 同步所需的硬件支持¶

    为解决数据竞争，需要硬件级的原子读写操作：

    • 原子操作要求“读-写”过程中，该内存位置不允许其他处理器访问；
    • 确保操作的“原子性”（不可分割性），避免中间状态被干扰。

    3. 原子操作的指令实现¶

    原子同步可通过单条指令或原子指令对实现：

    • 示例：寄存器与内存的“原子交换”指令（atomic swap）；
    • 或通过一对原子指令组合，保证操作的原子性。

    相关命令¶

    这一页讲解的是RISC-V架构中用于同步的两条关键指令：保留加载（Load Reserved）和条件存储（Store Conditional），它们配合实现原子操作，解决多处理器共享内存的同步问题，具体如下：

    1. 保留加载指令（Load Reserved: lr.d rd, (rs1)）¶

    • 功能：从寄存器rs1指定的内存地址加载数据到寄存器rd；
    • 关键特性：对该内存地址设置“保留标记”，用于后续检测该地址是否被其他处理器修改。

    2. 条件存储指令（Store Conditional: sc.d rd, (rs1), rs2）¶

    • 功能：将寄存器rs2中的数据存储到rs1指定的内存地址；
    • 执行结果判断：
    • 成功：若从lr.d执行后到sc.d执行前，该内存地址未被修改，存储成功，rd返回0；
    • 失败：若该内存地址的值已被修改，存储失败，rd返回非零值。

    这两条指令配合使用，可实现原子性的“读-改-写”操作，是RISC-V处理多处理器内存同步、避免数据竞争的核心硬件支持手段（例如用于实现互斥锁、原子变量更新等场景）。

    这里说一下递归函数应该怎么写¶

    有几个原则：

    1. 每次调用要准备2个位置 也就是sp=sp-16 一个用来放回去的位置x1 一个放回去的值

    3. 你在同一层执行的东西 不用管下面几层这个变量怎么变 就看这一层就好了。你要记住 写的对的话肯定会还原的。
    4. 一般递归的命令就是 jal x1 函数（比如fib fact）
